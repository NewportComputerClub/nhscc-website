Rules

  • Fresh code. Bringing an idea is fine, working on code beforehand is not. Please be fair :)
  • All team members present. No max/min team size, but all team members must be present for the majority of the event.
  • Don't hog the bandwidth. Respect others on the network, download large files and software at home.
  • Open source libraries, frameworks, etc. are good. Do not pirate or otherwise illegally use proprietary code!
  • Do not work on projects that are potentially dangerous to others or property in any way.
  • KnightHacks organizers reserve the right to disqualify any project at their discretion.




Judging and Prizes

  • Judging Criteria
  • Originality/Innovation. Does this hack create a solution to something yet to be solved?
  • Technical Difficulty. How difficult was this project for your team?
  • Completeness/Execution. We don't expect you to be completely finished, but the more the better!
